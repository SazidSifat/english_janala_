Difference between var, let, and const ?

LET

  • let is block-scoped,
  • let allows re-assignment but does not allow re-declaration
  • let is hoisted, but not initialized

CONST

  • const is block-scoped,
  • const allows neither re-declaration nor re-assignment,
  • const is hoisted, but not initialized

VAR

  • var is function-scoped,
  • var allows both re-declaration and re-assignment,
  • var is hoisted and initialized as undefined;
Difference between map(), forEach(), and filter()
In JavaScript, map(), forEach(), and filter() are array methods for different functions. The map() method is used to transform each element in an array and returns a new array with the transformed elements. It does not modify the original array. By contrast, forEach() only iterates through each item and does something with it, like logging or changing outside variables, but never returns a new array or a changed one. Last, filter() is applied to check every element against some condition and returns a new array containing only the elements that pass the test. All three of them are powerful, but used in different circumstances based on whether you want to change, do something with elements, or exclude items.
Explain arrow functions and how they are different from regular functions
Arrow functions in JavaScript are a more concise method of declaring functions. They are different from regular functions in that they use the => syntax and can utilize an implicit return if there is only one expression. For example, const add = (a, b) => a + b is significantly shorter than const add = function(a, b) { return a + b; }. Perhaps the most significant difference is the behavior of this. Arrow functions do not have their own this context; instead, they borrow it from the outer scope. In contrast, normal functions have their own this, which can lead to different behavior, especially when used within objects or event handlers. This makes arrow functions ideal for situations where you want to maintain the context of this from the outer function.
How JavaScript Promises work
A JavaScript Promise is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. It starts in a "pending" state and transitions to either "resolved" (fulfilled) or "rejected" based on whether the asynchronous operation succeeds or fails. Promises allow for more readable asynchronous code by using methods like then(), catch(), and finally(). The then() method is used to handle the resolved value when the promise is fulfilled, while catch() handles errors when the promise is rejected. The finally() method runs regardless of the promise's outcome, often used for cleanup tasks. This approach makes handling asynchronous code much cleaner and avoids the "callback hell" problem.
How closures work in JavaScript
109 wordsclear Humanize AI A JavaScript closure is a function that has access to its lexical environment even after the outer function that created it has ended. This means that an inner function can still utilize variables from its outer function, even if the outer function has ended. Closures are created when a function is declared within another function so that the inner function can "close over" variables and parameters of the outer function. This is useful for operations such as data encapsulation, where private variables are only accessible or modifiable through specific functions. Closures are employed in callbacks, event handling, and maintaining state between function calls.
